# Nivesh Platform Setup Scripts

This folder contains cross-platform setup scripts to initialize the Nivesh Platform from scratch on any machine.

## Quick Start

### Linux / macOS
```bash
chmod +x setup/setup.sh
./setup/setup.sh
```

### Windows (PowerShell — Recommended)
```powershell
powershell -ExecutionPolicy RemoteSigned -File setup\setup.ps1
```

Or if you have PowerShell 5+, you can just double-click `setup.ps1`.

### Windows (CMD — Basic)
```cmd
setup\setup.bat
```

## What These Scripts Do

All three scripts follow the same setup sequence:

### 1. **Prerequisites Check**
- Python 3.10+ (required)
- Node.js 18+ (required — will install via nvm/Homebrew/winget if missing)
- Docker (optional — only needed if you choose Docker PostgreSQL)

### 2. **PostgreSQL Setup**
Interactive choice:
- **[1] Docker** — auto-managed PostgreSQL 16 Alpine via Docker Compose (default)
- **[2] External** — use an existing PostgreSQL database (provide connection URL)

### 3. **ta-lib C Library** (setup.sh only)
- Ubuntu/Debian: `sudo apt-get install libta-lib-dev`
- macOS: `brew install ta-lib`
- Fallback: compile from bundled source in repo root

### 4. **Python Virtual Environment**
- Creates `backend/venv`
- Installs all dependencies from `backend/requirements.txt`

### 5. **Environment Files**
- `backend/.env` — API configuration + auto-generated SECRET_KEY
- `frontend/.env` — Frontend API URL (VITE_API_URL=/api/v1)

Both files are created if missing, or you can choose to overwrite existing ones.

### 6. **Database Migrations**
- Creates mutual fund tables (SQLAlchemy)
- Runs Alembic migration for stock market tables

### 7. **Optional Data Seeding** (Interactive)
#### Mutual Fund Data (30–60 minutes)
- Benchmark indices from CSV files
- Fund master records
- NAV history + metrics from AMFI API

#### Stock Market Data (20–40 minutes)
- Stock master (18 large-cap stocks + 3 indices)
- 5 years of OHLCV price history via yfinance

### 8. **Frontend Build**
- `npm install --legacy-peer-deps` in `frontend/`
- `npm run build` to create production bundle at `frontend/dist/`
- Backend automatically serves frontend from this directory

### 9. **Start API Server**
- Launches FastAPI/uvicorn on `http://localhost:8000`
- Frontend accessible at same URL (served by backend)
- API docs at `http://localhost:8000/docs`

## Environment Variables

### Backend (.env)
```
DATABASE_URL=postgresql+asyncpg://user:password@host:port/db
API_V1_STR=/api/v1
PROJECT_NAME=Nivesh API
ENABLE_AUTH=false                    # Set to true for production
SECRET_KEY=<auto-generated>          # Change in production
ACCESS_TOKEN_EXPIRE_MINUTES=30
```

### Frontend (.env)
```
VITE_API_URL=/api/v1                 # Same origin (backend serves frontend)
```

## Platform-Specific Notes

### Linux / macOS (`setup.sh`)
- Detects OS via `uname`
- Installs Node.js/nvm via curl if missing
- Installs ta-lib via apt-get (Ubuntu/Debian) or compiles from source
- Full colour output, strict error handling

### Windows PowerShell (`setup.ps1`)
- Requires PowerShell 5.0+
- Coloured output via `Write-Host`
- Node.js required (suggests `winget install OpenJS.NodeJS` or https://nodejs.org)
- ta-lib: attempts pre-built wheel; shows conda/WSL fallback if it fails
- Generates random SECRET_KEY

### Windows CMD (`setup.bat`)
- Runs on all Windows versions
- No colour output (CMD limitation)
- Recommends switching to PowerShell for better experience
- Node.js required (fatal error if missing)
- ta-lib: relies on pre-built wheels

## Common Issues

### Node.js not found
Install from https://nodejs.org (LTS recommended), or:
- macOS: `brew install node`
- Linux: `curl https://nodejs.org/... | bash`
- Windows: `winget install OpenJS.NodeJS` or download from nodejs.org

### TA-Lib compilation fails (Linux)
```bash
# Install pre-built
sudo apt-get install libta-lib-dev

# OR compile from bundled source
cd ta-lib
./configure && make && sudo make install && sudo ldconfig
```

### PostgreSQL connection fails
Ensure your database is reachable at the provided URL. If using Docker:
```bash
docker compose -f backend/docker-compose.yml logs postgres
```

### Frontend build fails
```bash
cd frontend
npm install --legacy-peer-deps
npm run build
```

Then restart setup script or manually start backend:
```bash
cd backend
source venv/bin/activate  # or venv\Scripts\activate on Windows
uvicorn app.main:app --reload --port 8000
```

### ENABLE_AUTH warning
The scripts default to `ENABLE_AUTH=false` (no authentication). For production, set to `true` and update your SECRET_KEY in `.env`.

## File Locations

After setup completes:
- Backend: `backend/venv/` (virtual environment)
- Backend config: `backend/.env`
- Frontend: `frontend/node_modules/`, `frontend/dist/`
- Frontend config: `frontend/.env`
- Logs: check console output or `backend/*.log`
- Database: PostgreSQL (Docker or external)
